## ISSUE-001: Order Management API auth/base URL conflict between spec and addendum
**Phase**: Phase 7
**Severity**: Blocker
**Description**: `project-context/product-spec.md` documents the order-lookup integration as: Base URL `https://orders.internal.company.com/v1`, auth via API key in an `X-API-Key` header. `project-context/addendum-notes-2026-07-08.md` (added to project-context after the original spec) states the Order Management API has since migrated to OAuth2 client-credentials auth: Base URL `https://orders-v2.internal.company.com`, token endpoint `/oauth/token`, bearer token on requests — and that the old API-key path is being sunset. These describe two different, mutually exclusive integration contracts (different base URL, different auth mechanism, and OAuth2 additionally requires token-acquisition/refresh handling that a static API key does not).
**Context**: Found while implementing `src/tools/order_lookup.py` for Phase 7 (order-lookup tool) — reached the point of building the HTTP client and needed to know the base URL and auth mechanism, and the two project-context files disagree. The addendum itself says "Not sure if this made it into the original product spec doc — flagging here so it's not missed," confirming the spec was not reconciled.
**Impact**: Cannot finalize the order-lookup tool's API client (base URL, auth header construction vs. OAuth2 token caching/refresh) without knowing which contract is authoritative. Building against the wrong one means the tool won't work against the real API and will need rework later — this also affects Phase 8 (response synthesis/citation) and any eventual integration test in Phase 9.
**Suggested resolution**: The addendum is dated after the spec (2026-07-08) and explicitly states the API-key path is being sunset, so it is likely authoritative — but this should be confirmed with the platform team rather than assumed, and `product-spec.md` should be updated so the two documents don't disagree going forward. Once confirmed, implement OAuth2 client-credentials with token caching/refresh in `_build_http_client()` in `src/tools/order_lookup.py`.
**Status**: Open
**Date**: 2026-07-09
